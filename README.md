## Overview

`nexa-report` contains the academic evidence for Nexa in the course Desarrollo de Aplicaciones Open Source. It documents the product problem, target segments, requirements, backlog, architecture, database design, implementation evidence, validation, deployment evidence, and annexes.

The report is maintained as Docs-as-Code in Markdown and aligns with the technical repositories `nexa-website`, `nexa-webapp`, and `nexa-platform`.

## Report Structure

| Section | Purpose |
| --- | --- |
| `front-matter/` | Cover, version history, table of contents, collaboration insights, student outcome evidence. |
| `10-chapter-1-introduction/` | Startup profile, solution profile, and target segments. |
| `20-chapter-2-requirements-elicitation/` | Competitors, interviews, needfinding, event storming, ubiquitous language. |
| `30-chapter-3-requirements-specification/` | User stories, impact mapping, and product backlog. |
| `40-chapter-4-product-design/` | Style guidelines, information architecture, UX/UI, DDD architecture, OOD, database design. |
| `50-chapter-5-implementation-validation-deployment/` | SCM, implementation evidence, validation interviews, deployment evidence, product video. |
| `annexes/` | Video, GitHub, deployment, validation, and controlled evidence annexes. |
